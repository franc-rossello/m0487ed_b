RA13 Diferencia els conceptes de codi font, objecte i executable.

Introducció al Desenvolupament d'un Programa Informàtic

El desenvolupament d'un programa informàtic passa per diverses etapes, cada una de les quals està relacionada amb la transformació de les idees i especificacions en codi executable. En aquest procés, hi intervenen diferents conceptes i eines que ajuden a transformar la nostra visió en un programa funcional. Alguns dels conceptes més importants són el codi font, el codi objecte i l'executable.

Conceptes Clau en el Desenvolupament d'un Programa

Codi Font

El codi font és l'estructura inicial d'un programa informàtic escrita en un llenguatge de programació de nivell alt. Aquest codi és comprensible per als humans i serveix com la base de tot el programa. Els llenguatges de programació com Python, Java, C++ i JavaScript són exemples de llenguatges en què es pot escriure codi font.

El codi font és, bàsicament, el conjunt d'instruccions i lògica que un programador crea per definir el comportament del programa. Aquest codi es conserva en fitxers d'extensió específica segons el llenguatge emprat (per exemple, .java per a Java, .py per a Python, .cpp per a C++).

Els programadors utilitzen eines com editores de text o entorns de desenvolupament integrats (IDE) per escriure el codi font, i poden incloure llibreries externes o API per afegir funcionalitat addicional.

Exemple de codi font (en Python):

def salutacio():
  print("Hola, món!")
  
salutacio()

Codi Objecte

Un cop el codi font s'ha escrit, és necessari transformar-lo en codi que l'ordinador pugui entendre directament. Aquesta transformació es realitza a través d'un procés anomenat compilació (en llenguatges com C o C++) o interpretació (en llenguatges com Python). El resultat d'aquest procés és el codi objecte, una versió binària del programa que encara no és executable.

El codi objecte és un conjunt de codi que el processador de l'ordinador pot llegir, però encara no té la capacitat d'executar-lo per si mateix. Aquest codi es genera a partir del codi font mitjançant un compilador o intèrpret. El codi objecte es pot associar a arxius amb extensions com .obj (en C/C++) o .o.

El codi objecte conté instruccions en codi màquina, però no és complet per si sol. Es pot combinar amb altres codi objecte i biblioteques per formar el programa final.

Exemple de codi objecte: (es representa com a dades binàries i no és llegible directament pel programador). Normalment, ve en forma d'arxius amb extensió .obj o .o generats per un compilador.

Codi Executable

Després de generar el codi objecte, és necessari realitzar un procés addicional anomenat enllaç (linking) per combinar tots els arxius de codi objecte en un sol arxiu executable. El resultat d'aquest procés és un executable, que és el fitxer final que es pot executar directament al sistema operatiu de l'usuari final.

L'arxiu executable conté el codi complet i preparat perquè el sistema operatiu el carregui i el faci funcionar. Aquest fitxer pot tenir extensions com .exe (en Windows) o no tenir extensió en sistemes Unix com Linux o macOS.

Una de les característiques més importants de l'executable és que, en contrast amb el codi font i el codi objecte, no es pot modificar ni llegir fàcilment, ja que està en un format binari que és comprensible només per l'ordinador. Un cop creat l'executable, el programa es pot executar sense necessitat de tenir accés al codi font original.

Exemple d'un arxiu executable en Windows: programa.exe

Procés de Desenvolupament d'un Programa Informàtic

El procés de creació d'un programa informàtic segueix diverses etapes que inclouen el desenvolupament, la compilació i l'enllaç. A continuació, es detallen aquestes etapes:

  1. Escriure el codi font: El programador crea el codi font que defineix la lògica del programa.
  2. Compilació: El compilador converteix el codi font en codi objecte. Aquesta etapa detecta errors sintàctics o lògics en el codi font.
  3. Enllaç (Linking): Els arxius de codi objecte es combinen amb llibreries externes per generar un programa complet.
  4. Creació de l'executable: El codi objecte es converteix en un fitxer executable que pot ser executat pel sistema operatiu.

Un cop completat el procés, el programa està llest per ser distribuït i executat pels usuaris.

Diagrama del procés de creació d'un programa informàtic: des del codi font fins a l'executable.

Conclusió

El desenvolupament d'un programa informàtic és un procés complex que implica diferents etapes, des de la creació del codi font fins a la generació de l'arxiu executable. Cadascun d'aquests passos és fonamental per assegurar que el programa sigui funcional i eficient. Entendre els conceptes de codi font, codi objecte i executable és clau per a qualsevol programador que vulgui comprendre el funcionament intern dels seus programes.
